## 📖 项目简介
本项目是一个基于 C/C++ 语言与 EasyX 图形库开发的“校园食堂菜品管理系统”。系统告别了传统的黑框控制台界面，采用现代软件工程的模块化思想，构建了完整的图形化人机交互（GUI）界面。

系统在底层**全面摒弃了定长数组，采用动态分配的单向链表**进行内存管理，结合本地二进制文件的双向持久化机制，实现了高效、安全的数据存取。项目内置了状态机驱动的无阻塞页面路由，并封装了外键级联校验等严谨的业务逻辑，提供从选餐、下单到售后评价的完整业务流转闭环。

## ✨ 核心特性与技术亮点

### 🛠 底层架构与算法优化
* **MVC 架构雏形**：将数据模型（单链表）、视图渲染（EasyX 绘图）与控制器（鼠标/键盘事件）严格解耦，代码结构清晰易维护。
* **状态机页面路由**：采用全局状态机变量无阻塞分发页面路由，结合双缓冲绘图技术（`FlushBatchDraw`），彻底消除画面闪烁。
* **内存链表与持久化机制**：采用头/尾插法动态构建五大核心链表（窗口、菜品、用户、订单、评价）。程序启动时利用 `fread` 从 `.dat` 二进制文件反序列化加载数据，关键操作时触发 `fwrite` 覆写回盘，兼顾高性能与数据安全。
* **GDI 字体渲染优化**：通过深入调用 Windows API (`LOGFONT`)，开启 ClearType 文本平滑技术，消除原生绘图函数的文字锯齿，实现商业级的高清抗锯齿 UI。

### 🛡️ 严谨的业务防御机制
* **外键级联拦截**：删除营业窗口前，系统探针会遍历菜品链表；若该窗口下仍有在售菜品，将严格拦截删除请求，防止产生“孤儿脏数据”。
* **高并发模拟与防超卖**：在学生点餐下单环节，加入严格的库存容量校验算法，防止出现越界购买和负数库存。
* **隐式密码保护**：在控制台输入密码时，通过底层接口捕获击键，不仅实现星号（`*`）掩码显示，还完美兼容并处理了 `Backspace` 退格逻辑。

## 🎯 业务功能模块

### 👨‍🎓 学生端模块
- **账号注册与登录**：支持新用户查重注册，双重密码校验。
- **菜品大厅**：分页浏览全局菜品。支持按名称的**模糊搜索**，按窗口/分类的**组合精确查询**，以及按价格/销量的**多关键字升降序重排**。
- **图形化下单**：动态展示菜品库存，一键下单并实时扣减库存、增加销量。
- **订单追踪与评价**：查看个人历史订单流转状态。对“已完成”的订单可进行星级打分与文字反馈，系统底层将自动重算并刷新该菜品的全局平均分。

### 👨‍💻 管理员后台模块
- **窗口与菜品维保**：全站物理窗口与菜品的 CRUD（增删改查）生命周期管理。
- **全站订单流转**：实时监控全局订单流，一键将待处理订单变更为发餐状态，驱动业务闭环。
- **可视化统计与报表**：
  - 动态遍历全站菜品销量，计算最大值以确定 Y 轴比例尺，利用图形库实时渲染直观的**销量柱状图**。
  - 支持将内存数据格式化对齐，一键导出为 `TXT 文本报表` 供打印存档。

## 🚀 安装与运行指南

### 环境依赖
1. **编译器**：推荐使用 Visual Studio (2019/2022) 或配置了 MinGW (GCC) 的 CLion/VSCode。
2. **图形库**：需提前安装 [EasyX Graphics Library](https://easyx.cn/)。

### 运行步骤
1. 将本项目源码（由于依赖 EasyX 的 `graphics.h`，请确保主文件后缀为 `.cpp`）克隆或下载到本地。
2. 在同一级目录下确保具备读写权限（系统首次运行时会自动生成 `windows.dat`, `dishes.dat` 等测试数据文件）。
3. 编译并运行主程序即可进入图形化主界面。
4. **默认测试账号**：
   - 管理员账号：`admin` / 密码：`123456`
